The module ships a **Convex tab for [Nuxt DevTools](https://devtools.nuxt.com)**. In dev, open DevTools (`Shift + Option + D` in the browser) and pick the Convex tab to inspect the running client:

- **Connection** — live WebSocket state: connected/disconnected, reconnect and retry counters, in-flight mutations and actions, plus the resolved deployment URL and a direct link to your [Convex dashboard](https://dashboard.convex.dev).
- **Queries** — every active query subscription with its arguments, listener count, update counter, current result, and the server-side log lines from that query's execution. The **source** button opens the Convex function's file in your editor.
- **Auth** — which auth adapter is active and, with Better Auth, the live `isLoading` / `isAuthenticated` / `isRefreshing` state.
- **Logs** — the Convex client's log stream (including verbose WebSocket lifecycle lines), filterable by level.

The panel is enabled by default whenever Nuxt DevTools is. To turn just the Convex tab off:

```ts [nuxt.config.ts]
export default defineNuxtConfig({
  convex: {
    devtools: false,
  },
})
```

::callout{icon="i-lucide-info"}
Everything DevTools shows is observed from the outside — the panel attaches to the same client your app uses without changing its behavior, opens no extra WebSocket, and none of it is included in production builds.
::

With Clerk or Auth0, the Auth tab shows the adapter but not its live state — those adapters provide their auth state at component level, out of the panel's reach.
