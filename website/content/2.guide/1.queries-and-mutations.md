All of these are auto-imported. Each also has a `useConvex*`-prefixed alias to avoid name clashes.

## `useQuery()`

Returns a `ComputedRef<T | undefined>` — `undefined` while loading — and **throws** query errors (caught by an `errorCaptured` boundary, mirroring React's render-throw). Args are reactive; pass `'skip'` to pause.

```vue
<script setup lang="ts">
import { api } from '#backend/api'

const messages = useQuery(api.messages.list, {})

const profile = useQuery(
  api.users.get,
  computed(() => userId.value ? { userId: userId.value } : 'skip'),
)
</script>
```

### `useQuery_experimental()`

The 1.39 result/error split — returns a `{ status }` union instead of throwing:

```ts
const result = useQuery_experimental(api.messages.list, {})
// result.value: { status: 'pending' }
//             | { status: 'success', data }
//             | { status: 'error', error }
```

## `useQueries()`

Subscribe to several queries over one subscription:

```ts
const results = useQueries({
  messages: { query: api.messages.list, args: {} },
  profile: { query: api.users.get, args: { userId } },
})
```

## `useMutation()` / `useAction()`

```ts
const send = useMutation(api.messages.send)
await send({ body: 'hi' })

const checkout = useAction(api.payments.checkout)
```

`useMutation` supports optimistic updates via the client.

## `usePaginatedQuery()`

Cursor pagination with a manual page-management engine:

```ts
const { results, status, loadMore } = usePaginatedQuery(
  api.messages.list,
  {},
  { initialNumItems: 20 },
)
```

A dual-overload `usePaginatedQuery_experimental` adds the object form and `throwOnError`.
