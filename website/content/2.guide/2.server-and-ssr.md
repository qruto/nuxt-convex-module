These server utilities are auto-imported in any Nitro server route, API handler, or SSR context. They mirror `convex/nextjs`.

## One-shot calls

```ts [server/api/messages.get.ts]
export default defineEventHandler(async () => {
  return await fetchQuery(api.messages.list, {})
})
```

`fetchQuery`, `fetchMutation`, and `fetchAction` run a single Convex call over HTTP (no subscription).

## Preloading for SSR

Preload on the server, hydrate on the client without a flash or a refetch:

```ts [server/...]
const preloaded = await preloadQuery(api.messages.list, {})
```

```vue [a component]
<script setup lang="ts">
const messages = usePreloadedQuery(preloaded)
</script>
```

`usePreloadedQuery` renders the preloaded value identically on server and initial client, then takes over as a live subscription — so there's no hydration mismatch.

`preloadedQueryResult(preloaded)` extracts just the value when you don't need the live handoff.

## Authenticated server calls

With the Better Auth integration installed, `backendAuth(event)` returns a request-scoped, authenticated Convex client:

```ts [server/api/me.get.ts]
export default defineEventHandler(async (event) => {
  const convex = backendAuth(event)
  return await convex.query(api.auth.getAuthUser, {})
})
```
