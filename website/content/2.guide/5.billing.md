Install `@convex-dev/polar` and the Vue ports of `@convex-dev/polar/react`'s components register automatically:

```bash
npm i @convex-dev/polar @polar-sh/checkout
```

Auto-registration is a Nuxt convenience — in a plain Vue app, import the components from `nuxt-convex-module/polar/vue` and register them yourself.

## `<CheckoutLink>`

Renders a Polar checkout link — embedded modal (default) or redirect — with optional lazy generation and trial configuration:

```vue
<template>
  <CheckoutLink :product-ids="[productId]" :trial-interval-count="7" trial-interval="day">
    Start free trial
  </CheckoutLink>
</template>
```

| Prop | Type | Default | Purpose |
| --- | --- | --- | --- |
| `product-ids` | `string[]` | required | Products offered at checkout |
| `embed` | `boolean` | `true` | Embedded modal; set `:embed="false"` to open the Polar checkout page instead |
| `lazy` | `boolean` | `false` | Generate the checkout URL on click instead of on mount |
| `theme` | `'dark' \| 'light'` | `'dark'` | Embedded checkout theme |
| `subscription-id` | `string` | — | Upgrade/change an existing subscription |
| `metadata` | `Record<string, string>` | — | Extra checkout metadata |
| `trial-interval` / `trial-interval-count` | `'day' \| 'week' \| 'month' \| 'year' \| null` / `number \| null` | — | Free-trial configuration |
| `locale` | `string` | — | Checkout locale |

## `<CustomerPortalLink>`

A link to the Polar customer portal (subscription management) — renders nothing until the portal URL resolves:

```vue
<template>
  <CustomerPortalLink>Manage subscription</CustomerPortalLink>
</template>
```

An optional `return-url` prop is forwarded to `generateCustomerPortalUrl` (the portal URL re-fetches when it changes). The rendered anchor opens in a new tab (`target="_blank"`).

Both components read the Polar action references (`generateCheckoutLink`, `generateCustomerPortalUrl`) from the auto-provided `api.billing` namespace — so unlike upstream's required `polarApi` prop, passing it is optional here. Use `:polar-api` to override.

::callout{icon="i-lucide-info"}
The Polar **backend component** (`@convex-dev/polar`) is configured in your Convex functions, exactly as upstream. This package ports only the framework-side components.
::
