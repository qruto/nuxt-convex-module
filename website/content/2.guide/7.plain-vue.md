Everything under `nuxt-convex-module/vue` is plain Vue 3 — no Nuxt required. The Nuxt module only automates the wiring below; in a bare Vue app (Vite, or any other setup) you do it yourself in `main.ts`.

Nuxt-only conveniences you give up: auto-imports (import every composable and component from `nuxt-convex-module/vue` instead), the [`#backend/*` aliases](/guide/import-aliases) (import your generated modules directly), the automatic client/api provide plugin, the Nitro server utilities from [Server & SSR](/guide/server-and-ssr), and the Better Auth middleware/proxy.

## Providing the client

The client travels through Vue's provide/inject under `ConvexClientKey` (an `InjectionKey<ConvexVueClient>`) — the Vue analog of `convex/react`'s `<ConvexProvider>`. Composables resolve it via `useConvex()` and throw if it's missing.

```ts [main.ts]
import { createApp } from 'vue'
import {
  ConvexVueClient,
  ConvexClientKey,
  provideBackendApi,
} from 'nuxt-convex-module/vue'
import { api } from '../convex/_generated/api'
import App from './App.vue'

const client = new ConvexVueClient(import.meta.env.VITE_CONVEX_URL)

const app = createApp(App)
app.provide(ConvexClientKey, client)
provideBackendApi(api, app)
app.mount('#app')
```

The WebSocket opens lazily on the first subscription. If you ever tear the app down, call `await client.close()` to dispose of it.

`provideBackendApi(api, app)` stores your generated `api` under `BackendApiKey` so the zero-argument conveniences work — `useBackendApi()` / `useBackendNamespace()` and the [Polar billing components](/guide/billing) defaulting to `api.billing`. It's optional: `useQuery(api.messages.list)` and friends take the function reference explicitly and don't need it. Pass the `app` argument when calling it from `main.ts`; without it, `provideBackendApi` uses `provide()` and must run inside a component's setup.

## Using the composables

Import them where Nuxt would have auto-imported:

```vue
<script setup lang="ts">
import { useQuery, useMutation } from 'nuxt-convex-module/vue'
import { api } from '../convex/_generated/api'

const messages = useQuery(api.messages.list)
const send = useMutation(api.messages.send)
</script>

<template>
  <ul>
    <li v-for="message in messages" :key="message._id">{{ message.body }}</li>
  </ul>
</template>
```

## Auth wiring

Inside a component's setup, `provideConvexAuth` is the whole story — it builds the reactive `ConvexAuthState`, wires the client's `setAuth`/`clearAuth` watchers, and provides the state under `ConvexAuthStateKey` for `useConvexAuth()` and the `<Authenticated>` / `<Unauthenticated>` / `<AuthLoading>` components below it:

```vue
<script setup lang="ts">
import { provideConvexAuth, useConvex } from 'nuxt-convex-module/vue'
import { useMyAuth } from './auth' // your auth provider's composable

const client = useConvex()
const { isLoading, isAuthenticated } = provideConvexAuth({ client, useAuth: useMyAuth })
</script>
```

Your `useAuth` composable returns `{ isLoading, isAuthenticated, fetchAccessToken }` (plus an optional `authVersion`). `fetchAccessToken` is an async function returning the JWT (or `null`); pass it as a `Ref` whose identity changes — or bump `authVersion` — when re-authentication is needed. See [Authentication](/guide/authentication) for the full contract.

### Outside a component

`provideConvexAuth` needs a setup context. To install auth state at the app level from `main.ts`, use `createScopedConvexAuthState` and provide the result yourself:

```ts [main.ts]
import { createScopedConvexAuthState, ConvexAuthStateKey } from 'nuxt-convex-module/vue'

const { state, scope } = createScopedConvexAuthState({ client, useAuth: myUseAuth })
app.provide(ConvexAuthStateKey, state)

// on teardown:
scope.stop()
```

It runs the auth watchers inside a detached `EffectScope` and returns it so you can dispose them with `scope.stop()`. The lower-level `createConvexAuthState(options, scope?)` builds the same state without providing it — but called outside a component setup with no scope, its watchers can never be disposed, so prefer the scoped variant there.
