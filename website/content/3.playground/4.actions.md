---
title: Actions
description: Call a Convex action with useAction — a one-shot promise into the Node runtime, with pending and error states handled in plain Vue.
---

Unlike queries, actions are **not reactive subscriptions** — `useAction` returns a
plain async function that resolves once per call. Actions can run in the Node
runtime and hit external services; this one computes text stats and a SHA-256
hash with ~600ms of artificial latency, so watch the pending state:

::playground-analyze
::

```vue
<script setup lang="ts">
import { api } from '#convex/api'

const analyze = useAction(api.analyze.text)

const input = ref('')
const result = ref()
const error = ref<string>()
const analyzing = ref(false)

async function submit() {
  error.value = undefined
  analyzing.value = true
  try {
    result.value = await analyze({ input: input.value })
  }
  catch (e) {
    error.value = e instanceof Error ? e.message : String(e)
  }
  finally {
    analyzing.value = false
  }
}
</script>
```

`analyze` has stable identity for the component's lifetime — pass it to children
or watchers freely. Because an action is just a promise, you own the in-flight
and error states: a `ref` flipped around `await` covers pending, and `try/catch`
catches server-side failures (empty input here returns zeros, but an external
API being down would land in the same `catch`).
