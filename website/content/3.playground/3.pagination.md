---
title: Pagination & optimistic updates
description: Grow a list page by page with usePaginatedQuery, and make writes feel instant with optimistic updates.
---

`usePaginatedQuery` turns a paginated Convex query into a growing list — ideal
for infinite scroll — while `withOptimisticUpdate` applies local writes to the
cached pages *before* the server confirms them. Add a task or toggle one and
watch it update with zero round-trip latency:

::playground-tasks
::

```vue
<script setup lang="ts">
import type { Id } from '#convex/dataModel'
import { api } from '#convex/api'
import { insertAtTop, optimisticallyUpdateValueInPaginatedQuery } from 'nuxt-convex-module/vue'

const { results, status, isLoading, loadMore } = usePaginatedQuery(
  api.tasks.listPaginated,
  {},
  { initialNumItems: 5 },
)

const addTask = useMutation(api.tasks.add).withOptimisticUpdate(
  (localStore, args) => {
    insertAtTop({
      paginatedQuery: api.tasks.listPaginated,
      localQueryStore: localStore,
      item: {
        _id: crypto.randomUUID() as Id<'tasks'>,
        _creationTime: Date.now(),
        text: args.text,
        completed: false,
      },
    })
  },
)

const toggleTask = useMutation(api.tasks.toggle).withOptimisticUpdate(
  (localStore, args) => {
    optimisticallyUpdateValueInPaginatedQuery(
      localStore,
      api.tasks.listPaginated,
      {},
      task => task._id === args.id ? { ...task, completed: !task.completed } : task,
    )
  },
)
</script>

<template>
  <label v-for="task in results" :key="task._id">
    <input type="checkbox" :checked="task.completed" @change="toggleTask({ id: task._id })">
    {{ task.text }}
  </label>
  <button :disabled="status !== 'CanLoadMore'" @click="loadMore(5)">
    Load more
  </button>
</template>
```

## Pagination statuses

`status` walks through four values: `LoadingFirstPage` while the initial page
streams in, then `CanLoadMore` when a `loadMore(n)` call would fetch another
page, `LoadingMore` while that fetch is in flight, and finally `Exhausted` once
the query has no more rows. `isLoading` is a shorthand for the two loading
states. `results`, `status`, and `isLoading` are `ComputedRef`s — destructure
them once and they stay reactive (use `.value` in script, bare in templates) —
while `loadMore` is a stable function you can call from anywhere.

## Optimistic updates

`withOptimisticUpdate` runs your handler against the local query cache the
moment the mutation is invoked, so the UI updates instantly: `insertAtTop`
prepends a placeholder item to the first loaded page, and
`optimisticallyUpdateValueInPaginatedQuery` maps over every loaded page's
items. The placeholder `_id`/`_creationTime` only exist locally — when the
server result syncs back, the optimistic write is rolled back and replaced by
the authoritative row (or simply rolled back if the mutation throws).
