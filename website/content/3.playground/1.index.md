Every page in this section embeds a **live demo** — real composables and components
from `nuxt-convex-module` talking to a real Convex deployment over the WebSocket
sync protocol. Nothing is mocked: what you see is exactly what your app gets.

## Running it locally

The playground uses an [anonymous local Convex deployment](https://docs.convex.dev/cli/local-deployments)
— no Convex account needed. From the repository root:

```bash [Terminal 1 — Convex deployment]
cd website
npx convex dev
```

```bash [Terminal 2 — docs site]
pnpm dev
```

The Convex functions live in [`website/convex/`](https://github.com/qruto/nuxt-convex-module/tree/main/website/convex)
— a small team-chat + tasks app exercising queries, mutations, actions, pagination,
and file storage. Each demo shows its own source, and the status pill in every
demo header reflects the live WebSocket connection state.

::callout{icon="i-lucide-shield-check"}
Authentication and billing need provider accounts (Better Auth against your own
deployment, Polar, Clerk, Auth0), so those demos run against demo stand-ins where
noted — the **components and composables are the real ones**. See the
[authentication guide](/guide/authentication) for full setup.
::

## Say hello

The classic Convex demo — `useQuery` streams the list live, `useMutation` writes.
Open this page in a second window and watch messages appear in both:

::playground-chat
::

```vue
<script setup lang="ts">
import { api } from '#convex/api'

const messages = useQuery(api.messages.list, {})
const send = useMutation(api.messages.send)

const body = ref('')
</script>

<template>
  <ul>
    <li v-for="message in messages" :key="message._id">
      <strong>{{ message.author }}</strong>: {{ message.body }}
    </li>
  </ul>
  <form @submit.prevent="send({ author: 'me', body }).then(() => (body = ''))">
    <input v-model="body" placeholder="Message">
    <button type="submit">Send</button>
  </form>
</template>
```

`messages` is a `ComputedRef` — `undefined` while the first result loads, then it
updates reactively on every change from any client. No refresh, no polling, no
cache invalidation.
