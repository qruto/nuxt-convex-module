---
title: Connection state
description: Watch the live WebSocket connection state with useConvexConnectionState — connectivity, retries, and in-flight request counters.
---

`useConvexConnectionState` exposes the Convex client's live `ConnectionState` —
the same object powering the status pill in every demo header. It updates
whenever any part of the state changes:

::playground-connection
::

```vue
<script setup lang="ts">
const connection = useConvexConnectionState()
</script>

<template>
  <dl>
    <template v-for="[key, value] in Object.entries(connection)" :key="key">
      <dt>{{ key }}</dt>
      <dd>{{ value }}</dd>
    </template>
  </dl>
</template>
```

The composable returns a `ShallowRef<ConnectionState>` with fields like
`isWebSocketConnected`, `hasEverConnected`, `connectionCount`,
`connectionRetries`, `hasInflightRequests`, `inflightMutations`,
`inflightActions`, and `timeOfOldestInflightRequest`. Fire the slow action
above and watch the in-flight counters tick. Note that upstream marks this
shape as unstable — it may gain fields, which also means the ref may update
more often over time.

::callout{icon="i-lucide-unplug"}
Try stopping `npx convex dev` — `isWebSocketConnected` flips to `false`,
`connectionRetries` starts climbing, and every status pill on this site turns
red. Restart it and watch the client reconnect on its own.
::
