---
title: Billing components
description: The Polar CheckoutLink and CustomerPortalLink components, auto-registered and wired to Convex actions.
---

When `@convex-dev/polar` is installed, `@qruto/nuxt-convex` globally registers
`<CheckoutLink>` and `<CustomerPortalLink>` — Vue ports of Polar's React
components. They call the `generateCheckoutLink` / `generateCustomerPortalUrl`
actions from your `api.billing` namespace and render plain `<a>` elements:

::playground-billing
::

```vue
<template>
  <!-- Auto-registered, no imports. Actions resolve from `api.billing`. -->
  <CheckoutLink :product-ids="['demo-product']" :embed="false" target="_blank">
    Buy demo product
  </CheckoutLink>

  <CustomerPortalLink return-url="/playground/billing">
    Manage subscription
  </CustomerPortalLink>
</template>
```

The components here are the **real published ones**, but the playground has no
Polar account behind it: the backend actions are **demo stand-ins** that
implement `@convex-dev/polar`'s exact action contract and return
`sandbox.polar.sh` URLs. That is also why the demo passes `:embed="false"` —
the sandbox URL is not a real Polar checkout, so it must open as a plain link
instead of the embedded overlay. With a real Polar setup you would keep the
default `embed`, and props like `trial-interval`, `metadata`, and `lazy` work
as upstream documents. See the [billing guide](/guide/billing) for the real
setup.
