---
title: Dynamic queries
description: Subscribe to a variable set of Convex queries at once with useQueries — add and remove subscriptions reactively without breaking the rules of composables.
---

`useQuery` handles one query. When the *set* of queries changes at runtime —
a dashboard with toggleable panels, a list of per-item subscriptions — you can't
conditionally call composables. `useQueries` takes the whole set as one reactive
object instead. Toggle the checkboxes to subscribe and unsubscribe live:

::playground-queries
::

```vue
<script setup lang="ts">
import type { RequestForQueries } from 'nuxt-convex-module/vue'
import { api } from '#convex/api'

const watchMessages = ref(true)
const watchTasks = ref(false)

// The request object is built in a computed — adding or removing a key
// subscribes or unsubscribes that query on the fly.
const request = computed(() => {
  const queries: RequestForQueries = {}
  if (watchMessages.value)
    queries.messageCount = { query: api.messages.count, args: {} }
  if (watchTasks.value)
    queries.taskStats = { query: api.tasks.stats, args: {} }
  return queries
})

const results = useQueries(request)
</script>

<template>
  <span v-if="results.messageCount === undefined">loading…</span>
  <span v-else-if="results.messageCount instanceof Error">
    {{ results.messageCount.message }}
  </span>
  <span v-else>{{ results.messageCount }} messages</span>
</template>
```

`useQueries` returns a `ShallowRef` of an object with the same keys as the
request: use `results.value[key]` in script, `results[key]` in templates. Each
key is `undefined` while its query loads, and if a query throws, the `Error` is
**returned as the value** for that key — not thrown — so one failing query never
takes down the others. Check `instanceof Error` before using a result.
