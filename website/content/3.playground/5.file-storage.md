---
title: File storage
description: Upload files to Convex storage with useUpload and useUploadQueue, then serve them back with useStorageUrl.
---

Three composables cover the whole Convex file-storage flow: `useUpload` for a
single file with reactive progress, `useUploadQueue` for batches with bounded
concurrency, and `useStorageUrl` to reactively resolve a served URL from a
storage id. Try a couple of small images:

::playground-files
::

```vue
<script setup lang="ts">
import { api } from '#backend/api'

const files = useQuery(api.files.list, {})
const save = useMutation(api.files.save)

// Single file: progress, error, and the resulting storage id as refs.
const { upload, isUploading, progress, error, storageId } = useUpload(
  api.files.generateUploadUrl,
  { onSuccess: (id, file) => save({ storageId: id, name: file.name, type: file.type, size: file.size }) },
)

// Resolve the served URL for the latest upload; the query is skipped
// while `storageId` is still null.
const latestUrl = useStorageUrl(api.files.url, storageId)

// Many files: enqueue a FileList, two uploads run at a time.
const { items, enqueue, progress: queueProgress } = useUploadQueue(
  api.files.generateUploadUrl,
  {
    concurrency: 2,
    onItemSuccess: (id, item) => save({ storageId: id, ...metaOf(item.file) }),
  },
)
</script>

<template>
  <input type="file" accept="image/*" :disabled="isUploading"
    @change="e => upload((e.target as HTMLInputElement).files![0])">
  <progress v-if="isUploading" :value="progress" max="1" />

  <input type="file" multiple @change="e => enqueue((e.target as HTMLInputElement).files)">
  <li v-for="item in items" :key="item.id">{{ item.status }} — {{ item.progress }}</li>

  <img v-if="latestUrl" :src="latestUrl">
</template>
```

`upload()` resolves to the new `StorageId` — or `null` on failure, with the
reason in `error`, so you never need a try/catch around it. The queue exposes
per-item `status`/`progress` plus aggregate refs, and saves each file the moment
its own upload succeeds. The gallery below the pickers is a plain `useQuery`
whose backend resolves URLs server-side, so new uploads and removals stream in
live on every connected client.
