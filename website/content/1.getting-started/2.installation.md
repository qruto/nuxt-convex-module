## 1. Add the module

```bash
npx nuxi@latest module add nuxt-convex-kit
```

```ts [nuxt.config.ts]
export default defineNuxtConfig({
  modules: ['nuxt-convex-kit'],
})
```

That's the whole setup. The composables, components, and server helpers are auto-imported. The module requires **Nuxt >= 4.1.0** and installs [`nuxt-security`](https://nuxt-security.vercel.app) as a module dependency — it applies Convex-aware CSP defaults (see [Security](/getting-started/configuration#security-csp)).

::callout{icon="i-lucide-info"}
Using **strict** pnpm? Add `public-hoist-pattern[]=@convex-dev/*` to `.npmrc` (or set `node-linker=hoisted`) so Convex can resolve component definitions.
::

## 2. Configure your deployment

```bash [.env]
NUXT_PUBLIC_CONVEX_URL=https://your-deployment.convex.cloud
NUXT_PUBLIC_CONVEX_SITE_URL=https://your-deployment.convex.site
```

The module reads these variables at build time, so put them in `.env` — the file Nuxt loads automatically. A custom file like `.env.local` only works if you pass it explicitly: `npx nuxi dev --dotenv .env.local`.

## 3. Run the app and Convex

```bash
npm run dev
npx convex dev
```

Read live data with `useQuery`, call `useMutation`, and resolve `#backend/api` from your generated Convex functions.

## Opt-in integrations

Install the packages you need — the module detects them and wires the integration automatically, no extra `modules` entries:

```bash
# Better Auth
npm i @convex-dev/better-auth better-auth @better-auth/passkey
# Clerk
npm i @clerk/vue
# Auth0
npm i @auth0/auth0-vue
# Billing components
npm i @convex-dev/polar @polar-sh/checkout
```

Force an integration on or off, or change the auth route:

```ts [nuxt.config.ts]
export default defineNuxtConfig({
  modules: ['nuxt-convex-kit'],
  convex: {
    // betterAuth: false, // or { authClient: './app/auth-client' }
    // clerk: false,
    // auth0: false,
    // polar: false,
    // authRoute: '/api/auth',
  },
})
```

`betterAuth` also takes an options object — point `authClient` at your own Better Auth client module to choose its plugins. See [module options](/getting-started/configuration) for details.
