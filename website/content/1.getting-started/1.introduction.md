`@qruto/nuxt-convex` brings [Convex](https://convex.dev) to [Vue](https://vuejs.org) and [Nuxt](https://nuxt.com). It is a **faithful port of Convex's official React/Next integration** — the same surface you'd get from `convex/react` and `convex/nextjs`, adapted to Vue with [VueUse](https://vueuse.org) conventions.

## What you get

- **Reactive data** — `useQuery`, `useQueries`, `useMutation`, `useAction`, and cursor `usePaginatedQuery`, backed by a live `ConvexVueClient`.
- **SSR & preloading** — `fetchQuery` / `fetchMutation` / `fetchAction` and `preloadQuery` + `usePreloadedQuery` for hydration-safe server rendering.
- **File storage** — `useUpload`, `useUploadQueue`, `useStorageUrl`.
- **Provider-agnostic auth state** — `useConvexAuth` / `provideConvexAuth` and the `<Authenticated>` / `<Unauthenticated>` / `<AuthLoading>` / `<AuthRefreshing>` components.
- **Opt-in integrations** — install `@convex-dev/better-auth`, `@clerk/vue`, `@auth0/auth0-vue`, or `@convex-dev/polar` and they wire themselves up.

::callout{icon="i-lucide-info"}
**Backend-only Convex components** (e.g. [`@convex-dev/resend`](https://github.com/get-convex/resend) for email) have no client integration to port — install them in your Convex backend and call their functions from Vue with the ordinary `useMutation` / `useAction` composables.
::

## Supported official packages

Each supported upstream package is ported with its **same public API**, adapted to Vue. Core Convex is always on; the rest auto-enable when their package is installed and are also importable from a dedicated subpath (`@qruto/nuxt-convex/<subpath>`).

| Official package | What you get | Enable | Subpath | Baseline |
|---|---|---|---|---|
| `convex` — `/react` + `/nextjs` | Composables, `ConvexVueClient`, auth helpers, SSR + server utils | always on | `/vue`, `/server` | `1.42` |
| `convex` — `/react-clerk` | Clerk auth — `provideConvexAuthFromClerk` · `<ConvexProviderWithClerk>` | `@clerk/vue` | `/clerk/vue` | `1.42` |
| `convex` — `/react-auth0` | Auth0 auth — `provideConvexAuthFromAuth0` · `<ConvexProviderWithAuth0>` | `@auth0/auth0-vue` | `/auth0/vue` | `1.42` |
| `@convex-dev/better-auth` | `useAuth`, `/api/auth/**` proxy, SSR prefetch, `auth` middleware, `<AuthBoundary>`, `backendAuth` | `@convex-dev/better-auth` | `/better-auth/vue`, `/better-auth/server` | `0.12` |
| `@convex-dev/polar` | Billing — `<CheckoutLink>` · `<CustomerPortalLink>` | `@convex-dev/polar` | `/polar/vue` | `0.9` |
| `@convex-dev/resend` | Backend-only — call via `useMutation` / `useAction` | — | — | — |

Everything is auto-imported under Nuxt; the subpaths exist for explicit/type-only imports — and `/vue`, `/clerk/vue`, `/auth0/vue`, and `/polar/vue` are self-contained, so they also work in a **plain-Vue (non-Nuxt)** app. (`/better-auth/vue`, `/better-auth/server` — where `backendAuth` lives — and `/server` rely on Nuxt-provided aliases/runtime config and need Nuxt.) See [authentication](/guide/authentication) and [billing](/guide/billing) for usage.

## Design principle: diffable with upstream

Every file mirrors its upstream origin so the port can track new Convex / Better Auth / Polar releases with a mechanical diff:

| This package | Upstream |
|---|---|
| `runtime/vue/**` | `convex/src/react/**` |
| `runtime/nuxt/**` | `convex/src/nextjs/**` |
| `runtime/clerk/**` | `convex/src/react-clerk/**` |
| `runtime/auth0/**` | `convex/src/react-auth0/**` |
| `runtime/better-auth/**` | `@convex-dev/better-auth/src/{react,nextjs}/**` |
| `runtime/polar/**` | `@convex-dev/polar/src/react/**` |

The full file map and pinned upstream baselines live in [`PARITY.md`](https://github.com/qruto/nuxt-convex/blob/main/PARITY.md).

## Reactivity, the VueUse way

The React hooks become Vue composables:

- Inputs accept `MaybeRefOrGetter` and are read with `toValue`, so args stay reactive.
- Derived results are `ComputedRef`; owned subscription state is a writable `ShallowRef`.
- Positional `useQuery` **throws** on error (caught by an error boundary, mirroring React's render-throw); the `_experimental` object form returns a `{ status }` union.
