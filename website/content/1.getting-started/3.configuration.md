## Module options (`convex`)

| Option | Type | Default | Description |
|---|---|---|---|
| `url` | `string` | `NUXT_PUBLIC_CONVEX_URL` | Convex deployment URL. |
| `siteUrl` | `string` | `NUXT_PUBLIC_CONVEX_SITE_URL` | Convex `.site` URL (HTTP actions / Better Auth endpoints). |
| `betterAuth` | `boolean \| { authClient }` | auto-detect | Force the Better Auth integration on/off, or pass `{ authClient: './path/to/module' }` to point at your own auth-client module (see [bring your own auth client](/guide/authentication#bring-your-own-auth-client)). Auto-enabled when `@convex-dev/better-auth` is installed. |
| `clerk` | `boolean` | auto-detect | Force the Clerk adapter on/off. Auto-enabled when `@clerk/vue` is installed. |
| `auth0` | `boolean` | auto-detect | Force the Auth0 adapter on/off. Auto-enabled when `@auth0/auth0-vue` is installed. |
| `polar` | `boolean` | auto-detect | Force the Polar components on/off. Auto-enabled when `@convex-dev/polar` is installed. |
| `authRoute` | `string` | `/api/auth` | Where the same-origin Better Auth proxy mounts. |

## Runtime config

The module publishes:

- `runtimeConfig.public.backend.url` / `.siteUrl`
- `runtimeConfig.backend.siteUrl`

Set them via environment variables (`NUXT_PUBLIC_CONVEX_URL`, `NUXT_PUBLIC_CONVEX_SITE_URL`) or the `convex` module options. The module resolves both at **build time** (module options win over the environment), and warns if no Convex URL is found.

## Import aliases

The module registers `#backend/*` aliases (Vite + Nitro) pointing at your Convex functions directory and its generated modules:

| Alias | Resolves to |
|---|---|
| `#backend/api` | `_generated/api` (`api`, `internal`, `components`) |
| `#backend/server` | `_generated/server` (`query`, `mutation`, `action`, …) |
| `#backend/dataModel` | `_generated/dataModel` (`DataModel`, `Doc`, `Id`, …) |
| `#backend/_generated` | `_generated` |
| `#backend` | your functions directory |

It also registers `#convex/auth-client` (Vite + Nitro, always — even with Better Auth off, so type-checking resolves): it points at your `betterAuth.authClient` module when set (absolute, or relative to the project root), otherwise at the bundled Better Auth client. The Better Auth runtime (`useAuth`, `<AuthBoundary>` typing, cross-domain helpers) resolves the client through this alias — see [bring your own auth client](/guide/authentication#bring-your-own-auth-client).

## Security (CSP)

The module bundles [`nuxt-security`](https://nuxt-security.vercel.app) as a module dependency and tightens a **Convex-aware Content Security Policy**: in production builds `connect-src` allows your deployment — and, when `siteUrl` is configured, the `.site` origin — over both HTTPS and WebSocket, while `img-src` / `media-src` allow Convex-served storage. The resource directives apply in `nuxt dev` too (nuxt-security enforces its default `img-src` during dev, which would otherwise block files served from Convex storage); `connect-src` is left untouched in dev so the Vite HMR and devtools WebSockets keep working. Your own directives are preserved — the Convex origins are appended.

The defaults apply only when the Convex URL is known at **build time** (`convex.url` or `NUXT_PUBLIC_CONVEX_URL` during the build), and setting `security: false` (or `security.headers.contentSecurityPolicy: false`) opts out entirely. Configure everything else through the standard `security` key.
