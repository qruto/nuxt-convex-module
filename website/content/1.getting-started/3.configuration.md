## Module options (`convex`)

| Option | Type | Default | Description |
|---|---|---|---|
| `url` | `string` | `NUXT_PUBLIC_CONVEX_URL` | Convex deployment URL. |
| `siteUrl` | `string` | `NUXT_PUBLIC_CONVEX_SITE_URL` | Convex `.site` URL (HTTP actions / Better Auth endpoints). |
| `betterAuth` | `boolean` | auto-detect | Force the Better Auth integration on/off. Auto-enabled when `@convex-dev/better-auth` is installed. |
| `clerk` | `boolean` | auto-detect | Force the Clerk adapter on/off. Auto-enabled when `@clerk/vue` is installed. |
| `auth0` | `boolean` | auto-detect | Force the Auth0 adapter on/off. Auto-enabled when `@auth0/auth0-vue` is installed. |
| `polar` | `boolean` | auto-detect | Force the Polar components on/off. Auto-enabled when `@convex-dev/polar` is installed. |
| `authRoute` | `string` | `/api/auth` | Where the same-origin Better Auth proxy mounts. |

## Runtime config

The module publishes:

- `runtimeConfig.public.backend.url` / `.siteUrl`
- `runtimeConfig.backend.siteUrl`

Set them via environment variables (`NUXT_PUBLIC_CONVEX_URL`, `NUXT_PUBLIC_CONVEX_SITE_URL`) or the `convex` module options.

## Import aliases

The module registers `#backend/*` aliases (Vite + Nitro) pointing at your Convex functions directory and its generated modules:

| Alias | Resolves to |
|---|---|
| `#backend/api` | `_generated/api` (`api`, `internal`, `components`) |
| `#backend/server` | `_generated/server` (`query`, `mutation`, `action`, …) |
| `#backend/dataModel` | `_generated/dataModel` (`DataModel`, `Doc`, `Id`, …) |
| `#backend/_generated` | `_generated` |
| `#backend` | your functions directory |

## Security (CSP)

The module bundles [`nuxt-security`](https://nuxt-security.vercel.app) and, in production builds with a known Convex URL, tightens a **Convex-aware Content Security Policy**: `connect-src` allows your deployment over HTTPS + WebSocket, and `img-src` / `media-src` allow Convex-served storage. Your own directives are preserved — the Convex origins are appended. Configure it through the standard `security` key.
